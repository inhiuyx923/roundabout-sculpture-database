# Roundabout Sculpture Database UI 规范

这份文档是之后制作页面、模块和新功能时共同遵守的视觉规范。目标不是做一个普通资料库，而是做一个带有“全球地图档案馆”气质的公共艺术数据库：冷静、精确、神秘、带一点夜间地图的荧光感。

## 1. 设计方向

名称：Dark Neon Atlas / 黑色荧光地图档案馆。

核心感受：
- 黑色与深海军蓝作为主体，像夜间打开的地图系统。
- 酸性黄绿色作为品牌信号，只用于关键焦点、当前状态、数字、标签和重要按钮。
- 蓝色作为地点、坐标、链接和普通点位的辅助色。
- 页面要像专业数据库和视觉档案馆，而不是营销官网、普通后台、Airtable 表格或传统博物馆页面。
- 图片区域未来必须承载真实街景截图或实拍图；没有真实图时，只能使用明确的数据占位视觉，不能假装成真实照片。

避免：
- 大面积渐变装饰、漂浮光球、过度插画感。
- 复古纸张、米色、暖棕、普通卡片式后台。
- 粗重科技字体。当前标题字体要保持轻、窄、冷静。
- 没有实际功能的控件。出现 Grid/List、筛选、排序等控件时，应尽量让它们真的可用。

## 2. 色彩

基础色：
- 背景：`#030609`
- 外壳：`#071018`
- 面板：`rgba(7, 15, 23, 0.94)`
- 柔和面板：`#0b1621`
- 卡片：`#08131d`

文字色：
- 主文字：`#f4f7f3`
- 次级文字：`#9aa7b3`
- 弱文字：`#65717d`

强调色：
- 品牌酸绿：`#d7ff00`
- 地点蓝：`#19a5ff`
- 有趣/精选点位橙红：`#ff6b2c`
- 低优先级灰：`#9ba0a4`

线条：
- 普通边线：`rgba(154, 175, 198, 0.18)`
- 高亮边线：`rgba(211, 255, 0, 0.62)`

使用原则：酸绿是“信号色”，不是背景色。一个屏幕里可以频繁看到它，但每次都应该出现在状态、按钮、数字、当前导航、地图点、收藏星标等有意义的位置。

## 3. 字体

标题字体：`Rajdhani`。
- 用于大标题、统计数字、需要地图系统感的显示文字。
- 默认使用 `300` 或 `400`，避免粗到像游戏标题。
- 大标题使用全大写，字间距保持 `0` 或非常轻微，不要拉得太开。

界面字体：`Inter`。
- 用于导航、筛选、按钮、卡片正文、表单和说明文字。
- 常规文字用 `400` 或 `500`。
- 标签、按钮、强调信息可用 `700` 或 `800`，但字号要小。

推荐字号：
- Hero 大标题：桌面 `62-92px`，移动端约 `48px`，行高约 `0.82-0.86`。
- 顶部导航：`13px / 500`。
- 筛选标签：`11px / 700-800`，全大写。
- 正文说明：`16px`，行高约 `1.55`。
- 统计数字：`27px / 400`，使用酸绿。
- 卡片标题：`16px / 700`。
- 卡片元信息：`11px`。

## 4. 页面结构

桌面端标准结构：
- 外层页面留 `18px` 呼吸空间。
- 主壳体是一个深色圆角容器，圆角约 `12px`。
- 顶部导航高度约 `74px`。
- 左侧筛选栏宽约 `350px`。
- 右侧内容区包含 Atlas 视图、地图、Archive 卡片区。

移动端标准结构：
- 去掉外层圆角，让页面全屏。
- 顶部导航精简为品牌、搜索/菜单图标。
- 筛选可以下移或做成抽屉式面板。
- 卡片两列起步；详情页以单列纵向阅读为主。

## 5. 组件规范

### 顶部导航
- 左侧是酸绿圆环 logo，加两行品牌文字。
- 中间导航采用文字链接：Atlas、Archive、Collections、Stories、About 等。
- 当前导航用酸绿下划线，不用大块背景。
- 右侧动作按钮要轻，边框细，不做厚重实心按钮。

### 筛选栏
- 标题区包含 Filters 与 Clear all。
- 每个筛选组都要有小号全大写标签。
- 输入框、下拉框、筛选 chip 都使用深色底和细边线。
- 当前选中的 chip 使用酸绿边线与浅酸绿背景。
- 底部结果按钮使用酸绿边框和酸绿文字，文字全大写，例如 `2,352 RESULTS`。

### Atlas Hero
- 左侧放大标题、项目说明、统计数字。
- 右侧放地图，地图是第一屏最重要的视觉对象。
- 地图点位使用发光小点，不使用默认地图图钉。
- 地图上可叠加数量 badge、缩放控件和区域强度图例。

### Archive 卡片
- 卡片应该是“图片优先”的结构：上方图片或截图，下方 metadata。
- 真实图片未接入前，可以使用数据占位视觉，但必须明显是占位，不伪造作品照片。
- 卡片右上角可保留星标，用来表达收藏/关注/重点。
- 标签颜色遵循数据含义：Curious 酸绿或橙红、Archive 蓝、Limited 灰。
- 卡片 hover 时只做细边线高亮，不要大幅放大或花哨动效。

### Detail 页面
未来详情页应遵循参考图 3：
- 左侧大图或图片组。
- 中间作品标题、地点、作者、年份、材料、编辑标记等 metadata。
- 右侧小地图卡片。
- 下方包含说明文字、来源、相关雕塑。
- 移动端先看图片，再看标题和关键 metadata，再看地图与相关内容。

## 6. 数据标签

目前数据里的颜色来自你之前的人工判断：
- 红色/橙色：你觉得很有意思，界面里可表达为 `Curious` 或重点点位。
- 蓝色：普通档案点位，界面里表达为 `Archive`。
- 灰色：有雕塑但截图困难、较弱或较无聊，界面里表达为 `Limited`。
- 紫色或其他：暂时保留为 `Other`，后续再清洗。

中文内部说明可以继续使用“有趣 / 普通 / 低优先级”，但公开界面目前使用英文标签，保持和参考图一致。

## 7. 交互规范

- 搜索、国家、城市、编辑标记筛选必须即时更新地图、数量和卡片。
- Clear all 必须恢复默认全部数据。
- 点击卡片应定位到地图上的对应点。
- Grid/List 这样的切换控件如果出现，就必须真的切换显示方式。
- 链接到外部网站时要打开新窗口，避免用户离开数据库页面。

## 8. 内容语气

公开界面语言：英文，保持国际数据库感。

语气关键词：precise, archival, obsessive, civic, public art, traffic-island sculpture, atlas, database。

避免太营销化的句子，比如 “Discover amazing sculptures today”。它应该像一个认真、冷静、带审美判断的档案项目。

## 9. 后续页面模块清单

后续新增页面应从这些模块中组合：
- Atlas 首页：全球地图、统计、筛选、精选卡片。
- Archive 页面：左侧筛选、右侧 Grid/List 数据浏览。
- Sculpture Detail 页面：图集、metadata、地图、来源、相关作品。
- Collections 页面：按国家、城市、形态、主题、年份、编辑标记组织专题。
- Stories/About 页面：项目背景、采集方法、Instagram 与 inhiu 入口。
- Admin/Data Review 页面：内部使用的数据清洗、截图状态、标签校对。

## 10. 发布与版本

- UI 规范修改本身不等于上线发布。
- 每次准备发布前，先给出版本号、变更文件、可见变化、风险说明，再由用户确认。
- 版本号格式为 `VYYMMDDHHMM`，例如 `V2605060228`。
- `CHANGELOG.md` 以后默认使用中文记录。
